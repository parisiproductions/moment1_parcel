Arbetsprocess

Automatisering med Parcel

För att automatisera webbplatsen använder jag Parcel, ett så kallat npm-paket, ett verktyg för att hantera och optimera filer. Jag börjar med att initiera npm i projektmappen, vilket skapar en package.json-fil för att hantera npm-paket och skript. Därefter installerar jag Parcel och lägger till det som en devDependency.

Jag skriver sedan skript i package.json för att automatisera utvecklingen och bygget av projektet. Med kommandot "start" startas en lokal utvecklingsserver för att testa koden, och med "build" skapas en optimerad version av projektet i en dist-mapp.

Under utvecklingen arbetar jag med källkod i src-mappen, och de genererade filerna i dist-mappen är optimerade html, css och JavaScript-filer färdiga för publicering. Jag gör en .gitignore-fil och skriver vilka filer som inte ska versionhanteras.

Versionshantering med GitHub

För versionshantering använder jag GitHub och skapar ett repository via Visual Studio Code. Jag använder en development-branch, för att testa ändringar innan de mergas till main-branch, som är kopplad till publicering.

Publicering med Render

Jag publicerar webbplatsen med Render, där jag har skapat två separata webbplatser. Den ena är kopplad till development-branch för testning, och den andra till main-branch för den slutgiltiga versionen. När jag är nöjd med ändringarna i development, mergas dessa till main, vilket automatiskt uppdaterar den publicerade webbplatsen.

Automatisering av bildoptimering

För att optimera bilder använder jag Parcel, som automatiskt ändrar storlek och konverterar bilder till moderna format som WebP. För mer kontroll använder jag även verktyget Sharp, där jag kan specificera bildkvalitet och inställningar för olika format i en konfigurationsfil. Jag testar optimeringsresultatet i Website Image Analysis Tool och justerar tills bilderna uppnår hög kvalitet och små filstorlekar (betyg A-B). Jag gör även test i Lighthouse för att se hur snabbt bilderna laddas.

Optimering av SVG-filer

Jag använder SVG-filer till min logotyp och till hamburgermenyn. SVG-filer optimeras med verktyget SVGO, som tar bort onödig metadata och whitespace, vilket gör filerna mindre och snabbare att ladda. Detta gör jag genom att köra SVGO via kommandotolken.

Bakåtkompatibilitet

För att säkerställa att webbplatsen fungerar i så många webbläsare som möjligt, inklusive äldre versioner, specificerar jag vilka webbläsare som ska stödjas i package.json. Jag anger att projektet ska fungera för populära och moderna webbläsare, men också vara bakåtkompatibelt för äldre versioner.
